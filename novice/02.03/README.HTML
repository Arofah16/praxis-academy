Mengenal paradigma Function Programming di JavaScript

paradigma functional pemrograman adalah gaya atau cara kita menulis program , sedangkan FUNCTIONAL PROGRAMMING adalah paradigma pemrograman yang berkutat pada komputasi yang terjadi didalam fungsi matematis dan menghindari terjadinya perubahan data, sedangkan fungsi matematis sendiri adalah sebuah hubungan antara input yang akan mengembalikan output

  1. function harus bersifat pure, dimana dia akan menerima parameter sebagai input,dan mengembalikan nilai , tanpa mengubah nilai dari variabel manapun
  2. function tidak boleh menggunakan nilai dari variabel lain yang bukan sebagai parameter
  3. function harus mengembalikan satu nilai untuk setiap input
  • SIDE EFFECT

side effect adalah dimana satu proses dapat mempengaruhi nilai sebuah variabel secara global dan menyebabkan error yang relatif sulit untuk di debug , penggunaan sintaks const dalam JavaScript ES6 yang kegunaanya adalah untuk menetapkan suatu nilai agar tidak berubah , namun hal tersebut tidak berlaku pada ARRAY dan OBJEK

  • IMMUTABILITY

adalah sebuah konsep dimana semua variabel ,setelah satu didefinisikan dandi assign sebuah nilai, nilai tersebut tidak boleh digantikan atau berubah, keuntunganya adalah kita terhindar dari side effect

  • HoF (Higher-Order-Function)

adalah sebah function yang menerima argument atau parameter berupa function contoh penggunaan Function programming adalah penggunaan FILTER

  • REKURSI

adalah sebuah fungsi yang memanggil dirinya sendiri

  • CURRYING

sebuah teknik dalam functional programming diman function tidak menambil semua argumen atau parameter dan digunakan sekaligus, melainkan mengembalikan sebuah function baru yang mengambil parameter kedua, yang kemudian mengembalikan function baru yang mengambil parameter ketiga, dan seterusnya hingga seluruh argumen atau parameter digunakan
