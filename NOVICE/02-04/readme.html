Memahami Secara Instan Asynchronous dan Synchronous

Secara sederhana Synchronous adalah proses pengeksekusian kode yang dijalankan secara berurutan berdasarkan kode yang tertulis.Dalam kasus Synchronous di atas, dapat disimpulkan, hasil yang didapatkan dari kode sesuai dengan urutan yang kita tulis. Berbeda dengan Synchronous, proses Asyncrhonous adalah proses pengeksekusian kode yang tidak sesuai dengan urutan yang ada atau bisa disebut menjalankan perintah selanjutnya tanpa menunggu perintah sebelumnya selesai.

Memahami Secara Instan Asynchronous dan Synchronous

  • Synchronous
  • Asyncrhonous

Synchronous

proses pengeksekusian kode yang dijalankan secara berurutan berdasarkan kode yang tertulis, dalam program ini kita menuliskan kode pada satu baris

anologi Synchronous

Proses Syncrhonous sama seperti proses antrian teller bank, di saat kamu berniat untuk menyetor/transfer uang melalui teller bank, kamu wajib mengikuti aturan yang dinamakan “antrian”, kamu belum bisa melakukan transaksi jika semua orang didepan kamu belum menyelesaikan transaksinya

Asyncrhonous

proses pengeksekusian kode yang tidak sesuai dengan urutan yang ada atau bisa disebut menjalankan perintah selanjutnya tanpa menunggu perintah sebelumnya selesai

analogii Asyncrhonous

bisa disamakan dengan sistem pemesanan pada restoran, Meja A memesan pada waktu 12.00, tanpa menunggu makanan Meja A tiba, Meja B bisa memesan makanan pada waktu 12.15, namun hasil yang didapatkan sesuai apa yang dipesan oleh Meja A dan Meja B. Jika Meja A memesan nasi goreng 2 piring, sementara Meja B hanya memesan nasi telor, maka Meja B bisa mendapatkan makanan lebih dulu dari pada Meja A

CALL BACK

Callback sebenarnya adalah function bedanya dengan function pada umumnya adalah di cara eksekusinya. Jika function pada umumnya di eksekusi berurutan dari atas ke bawah maka callback di eksekusi pada point tertentu, itu sebabnya di sebut callback, pada cara ini penggunaan callback dieksekusi dalam function lain melalui parameter

function dalam javascript adalah objek atau string yang disebut first class

  • function bisa dijadikan parameter
  • Function dapat disimpan ke dalam variabel
  • Seperti object pada umumnya, function bisa memiliki property dan method
  • Function dapat mengembalikan value dalam bentuk function

call back digunakan saat :

  • Injeksi atau modifikasi hasil eksekusi sebuah function
  • Event listener
  • Menangani proses asynchronous
